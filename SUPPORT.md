# Support

MySkills is public beta software. Support is best effort and focused on bugs, setup problems, security reports, and concrete product feedback.

## Supported Channels

- [GitHub Issues](https://github.com/jremick/myskills/issues): reproducible bugs, setup failures, documentation gaps, and specific feature requests.
- [GitHub private vulnerability reporting](https://github.com/jremick/myskills/security/advisories/new): suspected vulnerabilities, exposed secrets, authorization bypasses, package-safety escapes, or private-data exposure.

Do not post secrets, API tokens, `.env` contents, private package contents, production database URLs, object-storage keys, or sensitive logs in public issues.

## Current Support Scope

Supported for beta feedback:

- Fresh-clone setup and local self-hosting experiments.
- API, web, CLI, and MCP behavior documented in this repository.
- The example skill package under `examples/skills`.
- Release artifacts generated by the documented release workflow.

Not supported yet:

- Business-critical production operation.
- Hosted-service uptime commitments.
- Paid support or response-time SLA.
- Custom integrations outside the documented API, CLI, MCP, and package surfaces.
- Production migration guarantees beyond the compatibility and upgrade policies documented for the current prerelease.

## Issue Quality

Useful reports include:

- MySkills commit or release.
- Operating system and architecture.
- Node and npm versions.
- Docker/Postgres/object-storage setup when relevant.
- Exact command and sanitized output.
- Expected and actual behavior.

Security-sensitive reproduction details should go through the private vulnerability channel.
